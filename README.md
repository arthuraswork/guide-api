# 🧭 guide-api

REST API для туристического гида с **семантическим поиском** на базе нейросетей.  
Сервис позволяет подбирать достопримечательности и отели не только по жёстким фильтрам, но и по **свободному текстовому описанию**.


## 🛠 Стек

| Слой | Технологии |
|------|------------|
| Web-фреймворк | **FastAPI**, **Uvicorn** |
| Валидация | **Pydantic v2** |
| NLP / ML | **transformers**, **sentence-transformers** (модель `cointegrated/rubert-tiny2`), **scikit-learn** |
| Данные | JSONL + pickle, NumPy |
| Пакетный менеджер | **Poetry** (Python ≥ 3.12) |
| Скрипты | Bash (`./guideapi`, `./test`) |



## ✨ Ключевые возможности

- **Три режима поиска**: простой список, фильтрация по полям с операторами (`eq`, `neq`, `ge`, `le`), семантический поиск по тексту.
- **Предрасчёт эмбеддингов** (`sentence_precalc.py`) — модель грузится один раз при подготовке, в рантайме используется только косинусное сходство по матрице (быстро).
- **Автоактуализация конфигурации**: при старте сервис сам считает количество объектов в `.jsonl` и обновляет `data_config.json`, который используется для валидации `limit` в API.
- **HTTP-логирование** через middleware (успешные / проблемные запросы разделены маркерами в логах).
- **Fail-fast проверка данных** — `data_check.py` перед запуском проверяет, что все нужные файлы существуют.
- **Единый CLI** `./guideapi` для установки, запуска, дебага и smoke-тестирования.

## 🚀 Быстрый старт

```bash
# 1. Установка зависимостей
./guideapi install

# 2. Предрасчёт эмбеддингов (один раз, требует ~500 МБ для модели)
python3 sentence_precalc.py

# 3. Запуск production-сервера
./guideapi run          # на порту 8000
./guideapi run 8080     # на произвольном порту

# 4. Dev-режим с автоперезагрузкой
./guideapi debug

# 5. Запуск + автоматический smoke-тест
./guideapi test
```

> Перед любой командой `./guideapi` выполняет `data_check.py` — при отсутствии `.jsonl` файлов запуск прервётся с понятной ошибкой.  
> Шумные логи `transformers` / `huggingface_hub` подавлены переменными окружения (`TRANSFORMERS_VERBOSITY=error`, `HF_HUB_OFFLINE=1` и др.).



## 🌐 API

Все содержательные эндпоинты принимают `{category}` ∈ `places` | `hotels`.  
Интерактивная Swagger-документация доступна по `/docs` после запуска.

| Метод | URL | Описание |
|-------|-----|----------|
| `GET` | `/` | Карта API (также редирект с 404) |
| `GET` | `/{category}/list?limit=N` | Первые N объектов |
| `GET` | `/{category}/filter?param=...&op=...&val=...&limit=N` | Фильтрация по полю |
| `GET` | `/{category}/sentence?query=...&limit=N` | Семантический поиск |

### Примеры запросов

```bash
# Топ-5 отелей с рейтингом ≥ 4
curl "localhost:8000/hotels/filter?param=rate&op=ge&val=4&limit=5"

# Все отели со спа
curl "localhost:8000/hotels/filter?param=spa&op=eq&val=true&limit=10"

# "Тихий отель у моря со спа" — подбор по смыслу
curl "localhost:8000/hotels/sentence?query=тихий+отель+у+моря+со+спа&limit=3"
```

### Операторы фильтрации

| Код | Оператор |
|-----|----------|
| `eq` | `==` |
| `neq` | `!=` |
| `ge` | `>=` |
| `le` | `<=` |

| Поле | Тип | Ограничения |
|------|-----|-------------|
| `title` | `str` | 1–31 символ |

| `pool` | `bool` | `false` | Бассейн |
| `spa` | `bool` | `false` | Спа |



## 🧠 Как работает семантический поиск

1. **Офлайн** (`sentence_precalc.py`): для каждого объекта строится текст (`"{title} {type}: {description}"` для мест, расширенный булевыми признаками для отелей) и кодируется моделью `cointegrated/rubert-tiny2`. Эмбеддинги складываются в `.pkl`.
2. **Рантайм** (`sentence_compr.py`):
   - При старте все эмбеддинги загружаются в единую **NumPy-матрицу**.
   - Запрос пользователя кодируется той же моделью.
   - Через `cosine_similarity` из `scikit-learn` считается сходство запроса со всеми объектами → `np.argsort` → возвращаются top-K.

Предрасчёт позволяет отдавать результат за **десятки миллисекунд** — сама модель в рантайме вызывается только один раз (на кодирование короткого запроса).
